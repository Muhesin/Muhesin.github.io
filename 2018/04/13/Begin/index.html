前言

———本文记录本人建立个人博客的心路历程和感悟
  • Hexo是高效的静态站点生成框架，基于Node.js。
  • Node.js是基于Chrome V8引擎的JavaScript运行环境，
    可以在非浏览器环境下，解释运行JS 代码。
  • Git是目前世界上最流行的分布式版本控制系统，
  • github是面向开源及私有软件项目的托管平台，
    因只支持git作为唯一的版本库格式进行托管，故名gitHub。
  • Coding是面向开发者的云端开发平台,提供git/svn 代码托管,在线 WebIDE…

基本步骤

  • 基础环境搭建（node.js、git、github、coding配置）
    • node.js官网：https://nodejs.org/en/
    • git官网：https://git-scm.com/download/win
      检验配置：win+R；cmd，输入node -v,git –version
    • github配置：官网：https://github.com
      repository name:youename.github.io
      setting –> GitHub Pages 网址访问测试
      coding账户配置：官网： https://coding.net
      项目名称：youename.coding.me
      Pages服务->静态Pages->部署来源->master分支保存。网址访问测试
    • 注意事项:github与coding任一皆可，一起优点：保证访问速度。
      github与coding都有自己规范的项目命名规则，弄
      清楚可以避免不必要的麻烦（如http过长.配置时不方便）。
      • 用户名一旦确认不可更改；
      • 绑定解除繁琐
  • 二、安装Hexo
    • 任意位置建立新文件夹（用于存储本地文件）
      Git Bash执行命令
      -> npm install -g hexo
      -> hexo init //初始化
      -> hexo s //启动服务
      显示http：//localhost:4000,访问测试，成功则本地文件建立完成
  • 三、Hexo与Github、coding连接
    • 配置git个人信息
      Git Bash执行命令：
      设置user name和email
      -> git config –global user.name “yourname”
      -> git config –global user.email “youremail”
      生成秘钥
      -> ssh-keygen -t rsa -C “youremail”
      秘钥文件（.ssh->id_rsa.pub)复制，粘至Github、coding项目keys
    • 配置Deployment：
      文件根目录下修改_config.yml
  • 四、部署
    • Git Bash 文件根目录下执行命令：
      -> npm install hexo-deployer-git –save //部署扩展
      若没有执行者行命令，将会提醒deloyer not found:git
      -> hexo clean //清除缓存
      -> hexo g //生成静态文件
      -> hexo d //文件生成后立即部署
      访问检测 Github,coding 本地同步
  • 感悟
    • 遇到很多问题，反反复复重来，查资料，终于踏出了第一步，至此我会继续出发，停下容易迷茫，堕落终究消亡。
